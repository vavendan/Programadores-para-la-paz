El login es el proceso mediante el cual una aplicación verifica la identidad de un usuario antes de permitirle acceder a determinadas funciones o recursos. Para ello, generalmente se solicita un nombre de usuario o correo electrónico y una contraseña. Este mecanismo permite identificar quién está utilizando la plataforma y asignarle los permisos correspondientes según su rol. Algunas rutas deben estar protegidas porque contienen información privada, registros internos o funciones administrativas que no deben estar disponibles para cualquier persona. Restringir el acceso ayuda a prevenir el uso indebido de la información y reduce el riesgo de modificaciones no autorizadas. Además, proteger estas rutas fortalece la seguridad y la confianza de los usuarios en la plataforma. Por esta razón, no deben exponerse públicamente contraseñas, documentos de identidad, teléfonos, direcciones, tokens, claves de API ni cualquier otro dato personal o sensible de la comunidad.
